# 一、最关键参数（必须分析）

这些参数 **直接影响模型表达能力或学习过程**。

### 1 `lr`（learning rate）

```python
--lr
```

作用：

* 控制梯度更新步长

典型取值：

```
0.01
0.005
0.001
0.0005
0.0001
```

影响：

* 太大 → 不收敛
* 太小 → 收敛慢

**几乎所有论文都会分析**

这是一份非常典型的模型超参数调优（Hyperparameter Tuning）实验数据。数据对四个关键参数（`latdim`, `block_num`, `num_head`, `edgeSampRate`）进行了消融实验，每个参数值都跑了 10 次独立试验（Trial 0-9），并记录了推荐系统最常用的两个指标：**Recall（召回率）** 和 **NDCG（归一化折损累计增益）**。

为了让你更直观地看清趋势，我将这 10 次试验的数据进行了**取平均值**处理，并对这 4 个参数对模型性能的影响进行了详细分析：

### 1. `latdim` (节点嵌入维度 Embedding Size)
这个参数决定了 User 和 Item 初始特征向量的长度。

| `latdim` 值 | 平均 Recall | 平均 NDCG |
| :--- | :--- | :--- |
| **16** | **0.0748** | **0.0634** |
| 32 | 0.0694 | 0.0585 |
| 64 | 0.0619 | 0.0518 |
| 128 | 0.0606 | 0.0511 |

* **数据趋势：** 随着嵌入维度的**增加**，模型的 Recall 和 NDCG 呈现明显的**下降**趋势。16 维度时表现最好，128 维度时表现最差。
* **原理解析：** 这说明在这个特定的数据集上，模型面临着严重的**过拟合（Overfitting）**风险。高维度的 Embedding 赋予了模型过大的参数量和表达能力，导致它记住了训练集中的噪声，而失去了在测试集上的泛化能力。
* **结论：** 对于当前数据集，应该选择较小的潜层维度（如 **16**），既能达到最优性能，又能大幅节省内存和训练时间。

---

### 2. `block_num` (GNN+Transformer 层数/跳数)
这个参数决定了模型会进行几次“局部图聚合 + 全局注意力融合”的操作。

| `block_num` 值 | 平均 Recall | 平均 NDCG |
| :--- | :--- | :--- |
| 1 | 0.0624 | 0.0511 |
| 2 | 0.0700 | 0.0594 |
| 3 | 0.0732 | 0.0624 |
| **4** | **0.0767** | **0.0649** |

* **数据趋势：** 性能与层数呈**正相关**。层数从 1 增加到 4，Recall 和 NDCG 都有非常显著的提升。
* **原理解析：** 只有 1 层时，用户只能获取到直接交互过的物品信息（一跳邻居），视野受限。随着层数增加到 3 层或 4 层，GNN 能够捕捉到“用户的相似用户点过的物品”（高阶连通性），Transformer 也能在多次迭代中更精准地对齐全局特征。模型在第 4 层依然没有出现严重的“过平滑”（Over-smoothing，即所有节点特征趋同）现象，说明 Transformer 的加入和残差连接（Residual Connection）起到了很好的保护作用。
* **结论：** 应该选择较深的网络结构（**3 或 4 层**）。如果你有进一步算力，甚至可以尝试 5 层看看是否达到性能天花板。

---

### 3. `num_head` (Transformer 多头注意力头数)
决定了模型在计算自注意力时，会把特征切分成几个子空间来分别计算相似度。

| `num_head` 值 | 平均 Recall | 平均 NDCG |
| :--- | :--- | :--- |
| 1 | 0.0691 | 0.0584 |
| 2 | 0.0701 | 0.0596 |
| 4 | 0.0693 | 0.0584 |
| **8** | **0.0707** | **0.0600** |

* **数据趋势：** 各个头数下的表现**差距非常小**（Recall 基本都在 0.069 - 0.070 左右徘徊）。8 头和 2 头表现略好，但优势微乎其微。
* **原理解析：** 在自然语言处理（NLP）中，多头往往能捕捉不同的语法或语义关系。但在推荐系统的特征融合中，由于特征本身缺乏序列物理意义，多头带来的多子空间表征增益并不明显。这也说明 Transformer 的核心贡献在于全局的 Self-Attention 机制本身，而不是切分多少个 Head。
* **结论：** 这个参数对模型相对**不敏感**。为了平衡计算开销，建议选择 **2 或 4** 即可，没必要盲目追求 8 头（头数越多，矩阵乘法碎片化越高，推理更慢）。

---

### 4. `edgeSampRate` (边采样率)
这是一种类似 Dropout 的正则化手段（Edge Dropout），决定了在图卷积聚合时保留多少比例的边。

| `edgeSampRate` 值 | 平均 Recall | 平均 NDCG |
| :--- | :--- | :--- |
| **0.1** | **0.0700** | **0.0593** |
| 0.3 | 0.0695 | 0.0588 |
| 0.5 | 0.0685 | 0.0578 |
| 0.7 | 0.0701 | 0.0591 |

* **数据趋势：** 呈现出一种轻微的“U型”波动。保留率极低（**0.1**）或较高（**0.7**）时，性能稍微好一点，但在中间值（0.5）时略有下降。整体方差不大。
* **原理解析：** * 当采样率只有 **0.1** 时，相当于丢弃了 90% 的连边。这带来了极强的正则化效果（类似于强迫模型不要依赖特定几条边，而是学习更鲁棒的节点特征），从而取得了不错的泛化成绩。
    * 当采样率为 **0.7** 时，保留了大部分真实结构，信息完整度高，表现也不错。
* **结论：** 这是一个比较有趣的现象。由于之前分析 `latdim` 时我们发现模型容易过拟合，所以在这里采用极低的边采样率（**0.1 或 0.2**）作为一种强力正则化手段是一个明智的选择，同时丢弃大量边还能大幅加速 GNN 稀疏矩阵的计算速度。

### 💡 最终参数配置建议：
结合这份数据，如果要在该数据集上部署这个 TransGNN 模型获取最佳性能（SOTA），你应该在 `Params.py` 中这样设置：

* `--latdim 16` (防过拟合)
* `--block_num 4` (更深的特征交互)
* `--num_head 2` (兼顾速度与微弱的性能优势)
* `--edgeSampRate 0.1` (强力正则化，同时加快训练速度)
